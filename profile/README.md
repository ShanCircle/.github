# ShanCircle · 闪理山外

闪理山外是面向陕西理工大学校园场景的小程序与后端服务集合，当前重点覆盖微信登录、校内账号绑定、教务查询、校园网费/电费查询，以及支撑这些能力的 gRPC 服务、协议契约和容器化发布流程。

## 后端功能

- **统一入口服务**：`ShanCircle-Server` 使用 Go + Gin 提供 HTTP API，承担微信小程序登录、用户会话、SNUT 账号绑定/解绑、业务路由装配和对外接口聚合。
- **教务系统能力**：支持学期列表、今日/明日课程、学期课表、考试安排、成绩与绩点汇总等查询能力。
- **校园后勤能力**：支持校园网费查询、电费账号绑定、电表列表、电表详情、电表绑定与解绑。
- **SNUT 登录服务**：`SnutLoginService` 使用 Python gRPC 封装统一身份认证登录流程，处理登录页隐藏字段、密码加密、验证码识别，并返回登录结果与 Cookie。
- **HTTP 代理服务**：`HTTPProxyService` 统一管理短效 HTTP 代理，提供代理获取、缓存复用、失效上报和冷却淘汰能力，当前内置 `shenlong` provider。
- **服务契约管理**：`ShanCircle-protos` 集中维护 Protocol Buffers 定义，包含 SNUT 账号登录服务和 HTTP Proxy 服务，并提交 Go protobuf / gRPC 生成代码。
- **可观测性与安全边界**：后端统一使用结构化日志、`request_id` 链路标识、关键字段脱敏，避免在日志中暴露 token、openid、账号、客户端 IP 等敏感信息。
- **基础设施依赖**：主服务依赖 MySQL 保存业务数据，依赖 Redis 处理会话和缓存，并通过 gRPC 调用登录服务与代理服务。

## 前端功能

- **微信小程序客户端**：`ShanCircle-mp` 基于微信小程序 TypeScript / Sass 开发，使用 TDesign Miniprogram 组件库，支持自定义导航、深色模式和自定义 TabBar。
- **登录与账号管理**：通过微信登录获取用户会话，支持绑定/解绑校内统一身份认证账号，支持退出登录。
- **首页课程看板**：展示当前学期、当前周次、今日/明日课程，并支持每日 20:00 后自动切换明日课程。
- **教务服务页面**：提供学期课表、考试安排、成绩查询等入口；课表支持周次切换，考试支持批次筛选，成绩页展示学期汇总、学分和 GPA 信息。
- **后勤服务页面**：提供电费查询、网费查询；电费功能支持电费账号状态、电表绑定、列表查看、详情查询和解绑。
- **用户设置中心**：支持课程展示偏好、电费账号状态、联系客服、商务合作、关于我们等入口。
- **环境配置**：按微信小程序运行环境区分 `develop`、`trial`、`release` 配置，便于开发、体验版和正式版使用不同后端地址。
- **预留校园生活模块**：代码中保留快递代取、外卖代取、拼车、我的订单等页面与组件，后续可继续接入独立业务后端。

## 部署策略

- **服务拆分部署**：主 HTTP 服务、SNUT 登录 gRPC 服务、HTTP 代理 gRPC 服务独立构建和运行，主服务通过环境变量配置下游 gRPC 地址。
- **容器化交付**：三个后端服务均提供 Dockerfile；Go 主服务使用多阶段构建产出静态二进制，Python 服务预构建 wheel 后再进入运行镜像，降低运行阶段依赖和缓存污染。
- **GHCR 镜像发布**：后端仓库在推送 `v*` tag 时通过 GitHub Actions 构建并发布 `linux/amd64` 镜像到 GitHub Container Registry，同时生成 tag 与 sha 两类镜像标签。
- **CI 质量检查**：Go 服务执行 `go mod tidy`、`go vet`、race test、覆盖率统计、`golangci-lint` 和 `govulncheck`；Python 服务执行依赖安装、语法编译检查和 `pip check`；proto 仓库校验生成代码是否已同步提交。
- **私有依赖访问**：Go 主服务构建时通过 GitHub App token 读取组织内私有 proto 模块，Docker 构建使用 BuildKit secret 注入凭证，避免将 token 写入镜像层。
- **Docker Secret / Swarm 友好**：生产敏感配置通过 `/run/secrets/` 挂载，支持 `*_FILE` 环境变量读取微信 AppID/AppSecret、凭证加密密钥、代理 provider key/sign 等 secret。
- **环境变量驱动配置**：容器运行时通过环境变量覆盖端口、日志级别、MySQL、Redis、gRPC 地址、线程池、代理刷新策略等参数，适配本地调试、测试环境和生产环境。
- **最小权限运行**：运行镜像创建非 root 用户执行服务，减少容器运行时权限面。
